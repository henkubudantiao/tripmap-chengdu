# Btrip Design Spec (v8.0)

## HTML 布局（trip-map 样式）

### 地图
- Leaflet + CartoDB Light 瓷砖
- 高度 35vh，sticky top:0
- 标记：彩色圆形 + emoji（🍽️📍🏨🚃🍺）
- 路线：虚线 polyline
- 酒店：橙色标记，始终显示
- 导航弹窗：Apple Maps / 高德 / 百度

### Tabs
- sticky top:35vh
- 横向滚动，隐藏 scrollbar
- 胶囊形，选中蓝色白字
- 标签格式：D1·到达、D2·都江堰...

### 卡片时间线
- 左侧彩色圆点 + 虚线
- 右侧白色卡片，圆角 8px
- 时间（灰色小字）→ 名称（粗体）→ 描述（灰色）
- 点击展开详情
- Badge：餐（红）/景（蓝）/饮（紫）

### 详情区
- 小红书按钮：红色背景 #ff2442
- 导航按钮：蓝色背景
- 预约按钮：绿色背景
- 文字描述保留换行

### 颜色系统
- D1: #ff9500 (橙)
- D2: #007aff (蓝)
- D3: #af52de (紫)
- D4: #34c759 (绿)
- D5: #8e8e93 (灰)

## 数据结构

### 搜索数据（内嵌，用于模糊搜索）
```js
var NOTES=[{id:"...", t:"标题", d:"描述前200字", l:"点赞数", k:"作者", kw:"来源关键词"}];
var CMTS=[{id:"...", nid:"关联笔记id", c:"评论内容前150字", l:点赞数, n:"评论者"}];
```

### XHS 数组（页面展示用）
```js
var XHS=[{id:"...", title:"...", likes:"..."}];
```

### SHOPS 店铺推荐系统
```js
var SHOPS={
  区域id:[{
    name:'店名',
    desc:'一句话介绍',
    quote:'「小红书评论精选」',  // ⚠️ 必须有，原话引用
    tags:['分类标签'],
    dishes:'推荐菜品及价格',    // ⚠️ 餐馆必须列出推荐菜品
  }]
};
```

### DAYS 每日行程
```js
var DAYS=[{id:0,label:"总览",color:"#0071e3"},{id:1,label:"D1·到达",date:"...",title:"...",color:"#ff9500",locations:[...]}];
```

## 模糊搜索功能

### 搜索框
- 位于总览页最底部
- 无标题，直接是搜索框 + 提示文字
- placeholder: "搜索笔记标题、描述、评论内容..."

### 搜索结果展示

**笔记结果**：
- 类型标签（红色"笔记"）
- 标题（高亮关键词）
- 简介：单行显示，可左右滑动，自动滚动到关键词位置
- ❤️ 赞数按钮（红色胶囊，点击复制原文链接）
- 作者名

**评论结果**：
- 类型标签（蓝色"评论"）
- 评论内容（高亮关键词）
- ❤️ 赞数 + 评论者

### CSS 关键样式
```css
.sr-desc{
  display:block;
  white-space:nowrap;
  overflow-x:auto;
  -webkit-overflow-scrolling:touch;
  scrollbar-width:none;
}
```

### 事件绑定（手机兼容）
- 内联 `oninput="try{doSearch()}catch(e){}"` 兜底
- `addEventListener('input', doSearch)` 在 `go(0)` 内绑定
- `setTimeout` 100ms 后再绑一次

### JS 引号处理
- 所有 onclick 内嵌字符串用 `\x27`（十六进制单引号）
- 不要用 `\'`（在某些浏览器中会导致语法错误）

## 线路规划原则

### 住宿地址是核心锚点
- 用户给的每个住宿地址必须精确到门牌号
- 每天的行程围绕住宿位置展开，计算出发距离
- 不同住宿点之间的切换日要安排合理的交通

### 景点安排逻辑
- 地理位置要合理（不要来回折腾）
- 必去景点优先安排
- 商业化严重的商场类景点优先级低
- 节奏偏轻松逛吃，不要赶场

### 常见陷阱
- **天数算错**："16号下午到，20号高铁离开" = 5天（D1-D5），不是4天
- **漏掉整天**：日期跨度容易少算一天

## 依赖
- Leaflet 1.9.4 (CDN)
- CartoDB Light 瓷砖
- 无其他外部依赖

## 设计原则
- **数据驱动**：小红书高赞笔记 + 评论区实测数据
- **原话引用**：评论区精选保留原话，用「」包裹
- **菜品具体**：餐馆推荐必须列出具体菜品+价格
- **本地优先**：非连锁本地店 > 品牌连锁店
- **精简高效**：单文件 HTML，无构建工具
- **移动优先**：Apple 风格 UI，触控友好
- **可展开详情**：默认简洁，点击展开丰富内容
- **模糊搜索**：全量数据内嵌，支持中文模糊匹配
