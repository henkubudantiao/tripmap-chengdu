# Btrip Design Spec (v5.0)

## HTML 布局（trip-map 样式）

### 地图
- Leaflet + CartoDB Light 瓷砖
- **⚠️ Leaflet 必须内联（JS+CSS），CDN 在国内加载不了，会导致页面空白**
- 高度 35vh，sticky top:0
- 标记：彩色圆形 + emoji（🍽️📍🏨🚃🍺）
- 路线：虚线 polyline
- 酒店：橙色标记，始终显示
- 导航弹窗：Apple Maps / 高德 / 百度

### Tabs
- sticky top:35vh
- 横向滚动，隐藏 scrollbar
- 胶囊形，选中蓝色白字
- 标签格式：D1·到达、D2·都江堰...

### 卡片时间线
- 左侧彩色圆点 + 虚线
- 右侧白色卡片，圆角 8px
- 时间（灰色小字）→ 名称（粗体）→ 描述（灰色）
- 点击展开详情
- Badge：餐（红）/景（蓝）/饮（紫）

### 详情区
- 小红书按钮：红色背景 #ff2442
- 导航按钮：蓝色背景
- 预约按钮：绿色背景
- 文字描述保留换行

### 颜色系统
- D1: #ff9500 (橙)
- D2: #007aff (蓝)
- D3: #af52de (紫)
- D4: #34c759 (绿)
- D5: #8e8e93 (灰)

## 数据结构

### XHS 数组（v5 格式）
```js
const XHS=[
  {id:'6784dd5b000000000b0209ce',title:'太实用啦💟成都玉林路美食三刷全攻略',likes:'6863'},
  // id 是小红书笔记 ID，URL 通过 xhsUrl(id) 函数拼接
];
```

### location.xhs 数组
```js
xhs:[{id:'XHS数组中的id', label:'按钮显示文字'}]
```

### SHOPS 店铺推荐系统（v5 核心特性）
```js
const SHOPS={
  区域id:[
    {
      name:'店名',
      desc:'一句话介绍',
      quote:'「小红书评论精选」',  // ⚠️ 必须有，原话引用，用「」包裹
      tags:['分类标签'],
      dishes:'推荐菜品及价格',    // ⚠️ 餐馆必须列出推荐菜品
      url:'小红书笔记id'          // 关联的XHS笔记
    }
  ]
};
```

**SHOPS 质量要求**：
- 每个店铺**必须有 quote**（小红书评论区精选原话，带❤️点赞数）
- 餐馆**必须有 dishes**（推荐菜品 + 价格）
- 高赞优先，优先引用点赞数高的评论
- 非连锁本地店优先，商业化严重的低优先级

### shopToggle 展开组件
- 在景点卡片下方显示"📍 区域店铺推荐（N家）"的可展开列表
- 点击展开/收起店铺详情
- 每个店铺显示：名称、描述、评论引用、推荐菜品、小红书原文链接
- 通过 locCard 函数中的条件匹配自动关联（如"芳华街"匹配 SHOPS.fanghua）

### 预约信息格式
```
🎫 门票¥70
📱 预约方式：微信搜索"三星堆博物馆"小程序→选择日期购票
⚠️ 必须提前预约！
```

## 线路规划原则

### 住宿地址是核心锚点
- 用户给的每个住宿地址必须精确到门牌号（如"成华区双桥路249号附18号东城小居"）
- 每天的行程围绕住宿位置展开，计算出发距离
- 不同住宿点之间的切换日要安排合理的交通

### 景点安排逻辑
- 地理位置要合理（不要来回折腾）
- 必去景点优先安排
- 商业化严重的商场类景点优先级低
- 节奏偏轻松逛吃，不要赶场

### 常见陷阱
- **天数算错**："16号下午到，20号高铁离开" = 5天（D1-D5），不是4天
- **漏掉整天**：日期跨度容易少算一天
- **JS 语法错误**：time 值必须加引号（`time: '18:30'` 不是 `time: 18:30`）

## v5 vs 旧版(v2.1)的核心差异

| 特性 | v5 | v2.1 |
|------|-----|------|
| XHS数据 | 数组格式，id拼URL | 对象格式，完整URL |
| 店铺推荐 | SHOPS + shopToggle 展开组件 | 无 |
| 评论引用 | quote 字段，斜体引用样式 | 无 |
| 推荐菜品 | dishes 字段，带价格 | 无 |
| 复制链接 | copyBtn + toast 提示 | 无 |
| CSS变量 | 精简（8个颜色+sep+shadow+r） | 完整（20+变量） |
| 预约/预算 | 不含 | 含 reserve/budget/pay-summary |
| Leaflet | ⚠️ 必须内联 | CDN 加载 |

## 依赖
- Leaflet 1.9.4（⚠️ 必须内联，不能用 CDN）
- CartoDB Light 瓷砖
- 无其他外部依赖

## 设计原则
- **数据驱动**：小红书高赞笔记 + 评论区实测数据
- **原话引用**：评论区精选保留原话，用「」包裹
- **菜品具体**：餐馆推荐必须列出具体菜品+价格
- **本地优先**：非连锁本地店 > 品牌连锁店
- **精简高效**：单文件 HTML，无构建工具
- **移动优先**：Apple 风格 UI，触控友好
- **可展开详情**：默认简洁，点击展开丰富内容
